# Contributing to NOCFoundry

## Branch Model

Long-lived branches:

- `main`
- `release/*`
- `hotfix/*`

Short-lived branches:

- `sync/*`
- `feat/*`
- `fix/*`
- `chore/*`
- `refact/*`

## Pull Request Requirements

Every PR must include:

- Scope summary.
- Include/exclude path list.
- Verification commands and results.
- License and attribution impact statement.

For compliance-sensitive or third-party intake PRs, use templates in `docs/pr-templates/`.

## License and Attribution Guardrails

Do not add proprietary or trademarked assets that you do not have the right to distribute, including logos and brand visuals.

Any third-party code added to this repository must preserve Apache-2.0 header obligations, original notices when required, and accurate modification attribution.
New NOCFoundry-authored files should use one approved repository attribution form consistently.

Refer to:

- `docs/compliance/ATTRIBUTION_GATE.md`
- `docs/pr-templates/COMPLIANCE_PR.md`
